# 决策建议模块使用说明

## 📋 功能概述

决策建议模块是智能决策支持系统的核心功能之一，它能够基于数据分析自动生成智能决策建议，帮助您做出更好的管理决策。

## 🎯 主要功能

### 1. **查看决策建议列表**
- 页面会自动显示所有已生成的决策建议
- 每个建议卡片显示：
  - 优先级标签（高/中/低）
  - 建议标题和详细描述
  - 创建时间、分类、预期效果
  - 操作按钮（采纳/详情/拒绝）

### 2. **筛选建议**
- 使用顶部的下拉选择框可以按分类筛选建议：
  - 全部
  - 教学优化
  - 学生管理
  - 资源分配
  - 系统优化

### 3. **生成决策建议**

#### 方式一：快速生成（简单模式）
1. 点击"生成建议"按钮
2. 系统会自动使用默认配置生成 3 条建议
3. 生成的建议会立即显示在列表中

#### 方式二：条件配置生成（高级模式）
1. 点击"条件配置"按钮，打开配置对话框
2. 配置以下参数：
   - **建议类型**：选择要生成的建议类型（必填）
   - **优先级范围**：选择要生成的优先级（至少选择一个）
   - **数据来源**：选择分析的数据来源（可选）
   - **分析维度**：选择分析维度（可选）
   - **时间范围**：选择数据的时间范围（必填）
   - **建议数量**：设置要生成多少条建议（1-20条）
   - **算法模型**：选择生成算法（基于规则/机器学习/深度学习/混合模型）
   - **置信度阈值**：设置建议的置信度要求（0.5-1.0）
   - **特殊条件**：输入特殊要求（可选）
   - **自动执行**：是否启用自动执行（可选）

3. 配置完成后，有两种操作：
   - **保存配置**：仅保存配置，不生成建议
   - **生成建议**：根据配置立即生成建议

### 4. **管理决策建议**

#### 采纳建议
1. 点击建议卡片上的"采纳"按钮
2. 确认采纳操作
3. 建议会被标记为已采纳（可选：从列表中移除）

#### 查看详情
1. 点击建议卡片上的"详情"按钮
2. 查看完整的建议信息，包括：
   - 建议标题和描述
   - 分类和优先级
   - 创建时间和预期效果
   - 置信度（如果有）

#### 拒绝建议
1. 点击建议卡片上的"拒绝"按钮
2. 确认拒绝操作
3. 建议会从列表中移除

## 📊 数据存储

- 所有决策建议数据会自动保存到浏览器的 localStorage
- 刷新页面后数据不会丢失
- 数据在 `decision_suggestions` 键下存储

## 🔄 工作流程示例

### 场景：生成教学优化建议

1. **打开条件配置**
   - 点击"条件配置"按钮

2. **配置参数**
   - 建议类型：选择"教学优化建议"
   - 优先级范围：选择"高"和"中"
   - 数据来源：选择"学生表现数据"和"教师反馈数据"
   - 分析维度：选择"绩效分析"和"趋势分析"
   - 时间范围：选择最近30天
   - 建议数量：设置为 5 条
   - 算法模型：选择"混合模型"
   - 置信度阈值：设置为 0.8

3. **生成建议**
   - 点击"生成建议"按钮
   - 等待 2 秒左右，系统会生成 5 条建议

4. **查看和管理**
   - 查看生成的建议列表
   - 点击"详情"查看完整信息
   - 根据实际情况选择"采纳"或"拒绝"

## 💡 使用技巧

1. **快速生成**：如果只需要快速查看一些建议，使用"生成建议"按钮即可
2. **精确控制**：如果需要特定类型的建议，使用"条件配置"功能
3. **批量操作**：可以一次性生成多条建议，然后逐个查看和决策
4. **分类筛选**：使用分类筛选功能快速找到特定类型的建议

## ⚠️ 注意事项

1. 生成建议需要一定时间（通常 1-2 秒），请耐心等待
2. 拒绝的建议会从列表中永久移除，请谨慎操作
3. 建议数据保存在浏览器本地，清除浏览器数据会导致建议丢失
4. 条件配置中的必填项（建议类型、优先级范围、时间范围、建议数量）必须填写

## 🔧 技术说明

- 建议数据存储在 `src/store/decisionStore.js` 中
- 使用 Vue 3 Composition API 实现
- 数据持久化使用 localStorage
- 建议生成逻辑在 `generateConditionalSuggestions` 函数中

## 📝 建议数据结构

每个建议包含以下字段：
```javascript
{
  title: '建议标题',
  description: '建议详细描述',
  category: '建议分类（teaching/student/resource/system等）',
  priority: '优先级（高/中/低）',
  createTime: '创建时间',
  expectedEffect: '预期效果描述',
  confidence: '置信度（0-1之间）',
  value: '唯一标识符',
  label: '显示标签',
  adopted: '是否已采纳（true/false）',
  adoptedTime: '采纳时间'
}
```

## 🎯 采纳建议后的使用流程

### 采纳建议会发生什么？

当您采纳一个建议后，系统会：

1. **标记建议为已采纳**
   - 建议卡片会显示"已采纳"标签
   - 建议卡片会有绿色边框标识
   - 采纳按钮会变为"去使用"按钮

2. **记录到决策历史**
   - 建议会被自动添加到"决策历史"表格中
   - 显示采纳时间、决策场景、建议内容等信息
   - 初始状态为"进行中"

3. **更新场景统计**
   - 对应决策场景的"采纳数"会自动增加
   - 在决策场景卡片中可以看到更新的采纳数量

4. **可在智能决策页面使用**
   - 采纳的建议会保留在建议列表中
   - 在"智能决策"页面的"分析配置"区域
   - 可以在"决策建议"下拉框中选择已采纳的建议
   - 选择后，该建议会作为决策分析的配置项使用

### 如何使用已采纳的建议？

1. **方式一：从决策支持页面跳转**
   - 在已采纳的建议卡片上，点击"去使用"按钮
   - 系统会自动跳转到"智能决策"页面
   - 并自动选中该建议

2. **方式二：在智能决策页面选择**
   - 进入"智能决策"页面
   - 在"分析配置"区域的"决策建议"下拉框中
   - 选择您已采纳的建议
   - 然后进行决策分析

### 建议的完整生命周期

```
生成建议 → 查看详情 → 采纳建议 → 记录历史 → 在智能决策中使用 → 执行决策 → 评估效果
```
